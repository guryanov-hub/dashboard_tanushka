## Блок 2 — Лунний календар

### HTML id: `#moon-card`

Єдиний об'єднаний блок з трьома зонами: шапка з вибором дати, фазова смуга, вкладки.

---

### 2.1 Вибір дати

```html
<input type="date" id="moon-date" />
<button onclick="goToday()">Сьогодні</button>
```

**Стан:** `let selectedDate = new Date()` (встановлюється в 12:00:00 щоб уникнути UTC-зсуву).

**Логіка:**
- При зміні дати → `renderMoon()` перераховує все локально, без запитів
- `goToday()` скидає `selectedDate` до сьогодні та оновлює `input.value`
- `toDateVal(d)` → `d.toISOString().split('T')[0]`

---

### 2.2 Астрономічний розрахунок (локальний, без API)

```js
function calcMoonData(date) → { day, illum, waxing, signIdx, fraction }
```

**Алгоритм:**
1. Відома точка відліку: новолуння `2000-01-06T18:14:00Z`
2. Синодичний місяць: `29.53058867` діб
3. `fraction` = дробова частина кількості повних місяців від точки відліку (0..1)
4. `day` = `Math.floor(fraction * 29.53) + 1`
5. `illum` = `round((0.5 - 0.5 * cos(fraction * 2π)) * 100)` %
6. `waxing` = `fraction < 0.5`
7. Знак зодіаку: спрощена формула еклептичної довготи Місяця
   - `L = 218.316 + 13.176396 * (jd - J2000)`
   - `M = 134.963 + 13.064993 * (jd - J2000)` → переводиться в радіани
   - `lon = L + 6.289 * sin(M)`
   - `signIdx = floor(((lon % 360) + 360) % 360 / 30)`

**Точність:** похибка ~0.5–1 лунного дня. Достатньо для практичних рекомендацій.

Показується автоматично при:

## Автооновлення

```js
refreshAll();  // при старті
setInterval(() => { updateTime(); fetchWeather(); }, 60000);  // кожні 60 с
```

`renderMoon()` не потрапляє в setInterval — місяць оновлюється лише при зміні дати або перезавантаженні (знак зодіаку змінюється раз на 2–3 дні).

`fetchTennis()` викликається при старті та вручну кнопкою "Оновити" — не в setInterval, бо це платний API-запит.

## Блок 3 — Теніс WTA

### HTML id: `#tennis-card`

Розташований під `.main-grid` на повну ширину сторінки.

### Джерело даних

**Claude API** (`claude-sonnet-4-6`) з увімкненим інструментом `web_search_20250305`. Запит до `https://api.anthropic.com/v1/messages`. Без стороннього тенісного API — Claude сам шукає актуальні дані в інтернеті.

> ⚠️ У `dashboard.html` відкритому локально (через `file://`) запит може блокуватись браузером через CORS. Виджет гарантовано працює всередині Claude.ai як артефакт.

### Вкладки "Сьогодні / Завтра"

```html
<button id="ttab-0" onclick="setTennisDay(0)">Сьогодні</button>
<button id="ttab-1" onclick="setTennisDay(1)">Завтра</button>
```

Стан: `let tennisDay = 0`. Дані зберігаються в `tennisData` після першого завантаження — перемикання між вкладками не робить повторного запиту.

### Промпт до Claude API

```
Today is {today}, tomorrow is {tomorrow}.
Search for WTA singles matches for Ukrainian top-100 players on {today} and {tomorrow}.
Ukrainian players: Elina Svitolina, Dayana Yastremska, Marta Kostyuk, Angelina Kalinina, Lesia Tsurenko.
Return ONLY valid JSON: { "today": [...], "tomorrow": [...], "note": "", "source_info": "" }
```

### Версія для Claude.ai (артефакт/виджет)

Окрема розширена версія виджету будується через `visualize:show_widget` і має:
- Діапазон **7 днів** (сьогодні + 6 днів вперед) замість 2
- Групування матчів **по датах** із роздільниками
- **Фільтр по гравчинях** (пілюлі-кнопки: Усі / ім'я гравчині)
- Функції: `dateRange()`, `buildFilters()`, `setFilter(f)`, `renderMatches()`, `dayLabel(iso)`
- Промпт запитує `matches[]` з полем `date: YYYY-MM-DD`, сортованих за датою
